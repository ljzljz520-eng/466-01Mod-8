# Sky-Lark 详细设计文档

## 1. 数据库设计

系统采用 MySQL 8.0 作为主要关系型数据库。以下是核心表结构设计。

### 1.1 实体表（`record_entity`）

用于演示系统的基础增删改查功能。

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | PK, AI | 主键 ID |
| name | VARCHAR | 255 | NOT NULL | 实体名称 |
| description | VARCHAR | 255 | - | 实体描述 |

**建表语句（`schema.sql`）：**
```sql
CREATE TABLE IF NOT EXISTS record_entity (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255),
    description VARCHAR(255)
);
```

## 2. API 接口设计

所有 API 均位于 `/api` 路径下，由 Nginx 统一代理。

### 2.1 状态检查

- **GET** `/api/health`
  - **描述**：检查后端服务存活状态
  - **响应**：`{"status": "UP", "message": "Sky-Lark 后端服务运行中"}`

### 2.2 数据实体管理（`RecordController`）

- **GET** `/api/records`
  - **描述**：获取所有实体列表
  - **响应**：`[{"id": 1, "name": "...", "description": "..."}]`

- **POST** `/api/records`
  - **描述**：创建新实体
  - **请求体**：`{"name": "New Entity", "description": "Desc"}`
  - **响应**：创建成功的实体对象

## 3. 前端设计

### 3.1 页面路由

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `HomeView.vue` | 首页仪表盘，展示系统状态和数据列表 |
| `/about` | `AboutView.vue` | 关于页，展示项目信息 |

### 3.2 页面功能细化

#### 3.2.1 顶部导航
- **品牌区域**：显示项目名称与简短描述。
- **导航链接**：首页 / 关于，支持当前路由高亮。
- **响应式折叠**：小屏时导航自动换行或压缩布局。

#### 3.2.2 状态总览卡片
- **前端状态卡**：展示前端服务在线状态与更新时间。
- **后端状态卡**：请求 `/api/health`，显示服务可用性与异常提示。
- **数据库状态卡**：展示数据库连接信息与初始化提示。

#### 3.2.3 数据表格区
- **字段展示**：`ID / 名称 / 描述` 三列。
- **刷新按钮**：点击后重新拉取数据并显示加载状态。
- **空状态提示**：无数据时显示说明文本。

#### 3.2.4 交互反馈
- **加载状态**：请求中显示加载提示或骨架屏。
- **错误提示**：接口失败时显示可见提示信息。
- **按钮反馈**：按钮具备悬停与按压态反馈。

### 3.3 关键组件

- **AppLayout.vue**：全局布局组件，包含顶部导航栏与底部版权信息。
- **HomeView.vue**：核心视图，集成状态卡片与数据表格。

### 3.4 样式系统

- **Tailwind CSS**：负责全局原子化样式。
- **Element Plus**：提供标准 UI 组件（如 `el-table`、`el-card`、`el-button`）。

## 4. 安全设计

1. **CORS 策略**：后端配置 `WebMvcConfigurer`，允许来自前端容器的跨域请求（容器内网络环境）。
2. **环境隔离**：
   - 数据库密码通过 Docker 环境变量配置，避免硬编码在代码中（生产环境建议使用 Docker Secrets）。
   - 仅暴露必要端口（3000、8080），数据库端口 3306 可根据需要选择是否对外暴露。
