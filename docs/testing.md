# Sky-Lark 测试指南

## 1. 测试策略

本项目采用分层测试策略，确保从单元模块到整体系统的稳定性。

- **单元测试**：针对关键业务逻辑进行测试（当前阶段主要依赖 Spring Boot Test）。
- **集成测试**：验证 API 接口与数据库的交互。
- **系统测试**：通过 Docker Compose 验证全链路功能的连通性。

## 2. 手动验证清单

在提交代码前，请务必执行以下手动检查：

| 检查项 | 验证步骤 | 预期结果 |
|--------|----------|----------|
| **后端健康检查** | 访问 `http://localhost:8080/api/health` | 返回 `{"status":"UP"}` |
| **数据库连接** | 查看后端日志或访问演示数据接口 | 无 `SQLSyntaxErrorException` 或连接拒绝错误 |
| **前端页面加载** | 浏览器打开 `http://localhost:3000` | 首页加载迅速，无白屏，控制台无报错 |
| **状态卡片** | 观察三张状态卡片 | 状态正常、文案清晰、数据更新时间可见 |
| **刷新数据** | 点击“刷新数据”按钮 | 表格数据重新加载，按钮有加载状态反馈 |
| **空数据场景** | 清空表后刷新 | 显示空状态提示，不出现报错 |
| **错误提示** | 断开后端后刷新 | 页面展示错误提示，不崩溃 |
| **移动端适配** | F12 切换至手机模式（如 iPhone SE） | 卡片与表格纵向排列，无横向滚动条 |

## 3. 自动化测试

### 3.1 后端测试

运行 Maven 测试命令：
```bash
cd backend
mvn test
```

### 3.2 API 冒烟测试

可以使用 `curl` 快速验证核心接口：

```bash
# 检查健康状态
curl -s http://localhost:8080/api/health

# 获取演示数据
curl -s http://localhost:8080/api/records
```

## 4. 常见问题排查

- **后端启动较慢**：Spring Boot 首次启动初始化 JPA 可能需要 10-20 秒，请耐心等待健康检查端口变绿。
- **数据库拒绝连接**：请检查 `docker-compose.yml` 中 `db` 服务的健康检查配置，确保后端在数据库完全就绪后才尝试连接。
